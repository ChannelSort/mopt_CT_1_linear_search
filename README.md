# Методы одномерного поиска

Лабораторная работа по методам одномерного поиска. Проект реализован как расширяемая OOP-инфраструктура для численных экспериментов, построения графиков, выгрузки таблиц и подготовки данных для отчета.

## Что реализовано

- Все методы, перечисленные в условии лабораторной:
  - `Passive Search`
  - `Dichotomy`
  - `Golden Section`
  - `Fibonacci`
  - `Parabola`
  - `Brent (SciPy)`
- Все требуемые типы функций:
  - одна "хорошая" унимодальная функция;
  - две специальные унимодальные функции;
  - одна мультимодальная функция;
- инфраструктура эксперимента:
  - запуск всех комбинаций `функция x метод x epsilon`;
  - подсчет числа итераций и вызовов функции;
  - накопление истории интервала неопределенности;
  - сохранение таблиц в `.csv`;
  - сохранение графиков в `outputs/plots`;
  - отдельная стратегия для мультимодальной функции: `разведка + активный метод`.

## Модели функций

- `GoodUnimodalFunction`
  - квадратичная функция `(x - 2)^2 + 1`;
  - удобна как базовый эталон для сравнения методов.
- `PlateauAsymmetricUnimodalFunction`
  - функция с плато около минимума и асимметричным ростом;
  - полезна для проверки устойчивости методов.
- `SecondSpecialUnimodalFunction`
  - асимметричная функция `exp(-x) + |x - 0.5|^1.5`;
  - показывает поведение методов на негладкой и несимметричной форме.
- `MultimodalFunction`
  - модифицированная функция Растригина;
  - используется для эксперимента с предварительной разведкой.

## Методы оптимизации

- `PassiveSearchOptimizer`
  - реализован векторизованно через NumPy;
  - точность выбирается в соответствии с конспектом через условие `2h <= eps`.
- `DichotomyOptimizer`
  - реализует классическую схему двух симметричных точек около середины интервала.
- `GoldenSectionOptimizer`
  - использует переиспользование одного значения функции между соседними шагами.
- `FibonacciOptimizer`
  - использует отношения чисел Фибоначчи для сужения интервала.
- `ParabolaOptimizer`
  - реализует метод параболической интерполяции;
  - содержит защиту от некорректного параболического шага.
- `BrentOptimizer`
  - реализован через `scipy.optimize.minimize_scalar(..., method="brent")`;
  - используется как библиотечный эталон.

## Архитектура

Ключевые сущности проекта:

- `ObjectiveFunction`
  - инкапсулирует целевую функцию;
  - считает число вызовов;
  - поддерживает одиночные и векторизованные вычисления.
- `OptimizationResult`
  - единый контейнер результата;
  - хранит минимум, число итераций, число вызовов, историю интервала и статус.
- `Optimizer`
  - общий интерфейс для всех методов.
- `IterativeOptimizer`
  - базовый класс для интервальных методов;
  - унифицирует сбор истории, критерий остановки и сообщения о завершении.
- `OptimizationExperiment`
  - запускает серию экспериментов;
  - формирует таблицы;
  - строит графики;
  - сохраняет результаты на диск.

Дополнительно используются:

- декоратор `validate_bounds` для валидации входных параметров;
- `temporary_bounds(...)` для временного сужения интервала;
- `run_multimodal_strategy(...)` для сценария `разведка + уточнение`.

## Что генерируется при запуске

Команда:

```bash
python lab1_linear_search.py
```

Что делает программа:

1. Запускает все методы на всех функциях и всех значениях `eps`.
2. Выводит сводную таблицу в консоль.
3. Сохраняет сводную таблицу в `outputs/tables/experiment_summary.csv`.
4. Сохраняет отдельные таблицы по каждой унимодальной функции.
5. Строит графики:
   - `epsilon -> iterations`;
   - `epsilon -> function calls`;
   - динамика интервала неопределенности.
6. Выполняет отдельный мультимодальный эксперимент и сохраняет таблицу сравнения стратегий.

## Важные замечания

- `Passive Search` может быть очень дорогим по числу вычислений при малых `eps`.
- Для `Passive Search` при достижении лимита числа узлов результат помечается как предупреждение, а не как полная сходимость.
- `Brent (SciPy)` использует переданный интервал как начальный `bracket`, а не как жесткие границы поиска.
- Стандартный интерфейс `SciPy` не предоставляет историю внутренних интервалов для метода Брента, поэтому соответствующий график для него не строится.
- Для мультимодальной функции прямой запуск активного метода может привести к локальному минимуму; поэтому добавлена отдельная двухэтапная стратегия.
- Полный запуск экспериментов оформлен через отдельную функцию `main()`, что делает код чище и удобнее для повторного использования.
