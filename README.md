# mopt_CT_1_linear_search

Лабораторная работа по методам **одномерной оптимизации** (linear search methods) с акцентом на сравнение эффективности алгоритмов.

Проект реализован в виде **расширяемого OOP-фреймворка** для проведения экспериментов, визуализации результатов и генерации данных для отчёта.

---

## Возможности проекта

* Унифицированный интерфейс для всех методов оптимизации
* Поддержка **унимодальных и мультимодальных функций**
* Подсчёт числа вызовов функции (важно для анализа эффективности)
* Сбор полной истории сужения интервала (для большинства методов)
* Автоматический запуск серии экспериментов
* Построение графиков:

  * зависимость итераций от `ε`
  * зависимость числа вызовов функции от `ε`
  * динамика сужения интервала
* Экспорт таблиц в `.csv` для отчёта
* Поддержка двухэтапной стратегии (разведка + уточнение)

## Реализованные модели функций

### Унимодальные:

* **GoodUnimodalFunction** — простая квадратичная функция
* **PlateauAsymmetricUnimodalFunction** — плато + асимметрия
* **SecondSpecialUnimodalFunction** — экспонента + степень (сложная форма)

### Мультимодальная:

* **MultimodalFunction (Rastrigin)** — много локальных минимумов

---

## Реализованные методы оптимизации

### Полностью реализованы:

* **Passive Search** (пассивный перебор, векторизованный)
* **Brent (SciPy)** — через `scipy.optimize.minimize_scalar`
* **Dichotomy**
* **Golden Section**
* **Fibonacci Search**
* **Parabolic Interpolation**

---

## Архитектура

Проект построен вокруг следующих ключевых абстракций:

* `ObjectiveFunction` — функция с подсчётом вызовов
* `Optimizer` — базовый интерфейс метода
* `IterativeOptimizer` — общий каркас для интервальных методов
* `OptimizationResult` — результат работы алгоритма
* `OptimizationExperiment` — менеджер экспериментов

Дополнительно:

* генераторы (`_generate_steps`) для пошаговой логики методов
* декораторы для валидации входных параметров
* context manager для временного изменения интервала

## Запуск экспериментов

```bash
python lab1_linear_search.py
```

Что произойдёт:

1. Запустятся все комбинации:
   * функций
   * методов
   * значений `ε`
2. Выведется таблица в консоль
3. Сохранятся:

   * CSV таблица (`outputs/tables`)
   * графики (`outputs/plots`)
4. Построятся графики динамики интервала

---

## Пример эксперимента

Используются значения точности:

```python
epsilons = [1e-1, 1e-2, ..., 1e-8]
```

Для каждой комбинации сохраняются:

* число итераций
* число вызовов функции
* найденный минимум
* статус сходимости

## Мультимодальная стратегия

Реализована двухэтапная схема:

1. **Разведка (Passive Search)**
2. **Уточнение (любой активный метод)**

Сравнивается с:

* тем же методом **без разведки**

Функция:

```python
run_multimodal_strategy(...)
```
